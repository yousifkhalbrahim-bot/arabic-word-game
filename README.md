# لعبة الكلمات العربية 🎮

لعبة كلمات عربية متعددة اللاعبين بنظام ساعة الشطرنج. كل لاعب يلعب من جواله، والكلمات تتزامن لحظياً عبر الإنترنت.

## ✨ المميزات

- **لعب أونلاين**: كل لاعب على جواله، اتصال لحظي عبر Supabase Realtime
- **نظام غرف بكود**: 4 أرقام يكفي للانضمام
- **3 سرعات**: بوليت (دقيقة)، بليتز (5 دقائق)، رابيد (10 دقائق)
- **10 نهايات جاهزة + نهاية مخصصة**
- **قاموس عربي مدمج** (600+ كلمة) + قبول كلمات إضافية بالاتفاق
- **وضع اللعب على جهاز واحد** كاحتياط
- **تصميم عصري** بنظام ساعة الشطرنج بتدرجات الذهبي والفيروزي

## 🚀 خطوات التشغيل والنشر

### 1️⃣ الإعدادات المحلية

```bash
# في مجلد المشروع
npm install
cp .env.local.example .env.local
# عبّي القيم في .env.local (راح نجيبها من Supabase في الخطوة التالية)
```

### 2️⃣ إنشاء قاعدة البيانات (Supabase)

1. روح على [supabase.com](https://supabase.com) وسوّ حساب (مجاني)
2. اضغط **New Project**:
   - اختر اسم (مثلاً: `arabic-word-game`)
   - اختر باسوورد قوي للداتابيس (احفظه عندك)
   - اختر أقرب منطقة جغرافياً
3. انتظر دقيقتين لين يجهز المشروع
4. روح على **SQL Editor** من القائمة الجانبية
5. اضغط **+ New Query**
6. الصق محتوى ملف `supabase-schema.sql` كامل
7. اضغط **Run** (أو Ctrl+Enter)
8. لازم تشوف رسالة "Success"

### 3️⃣ نسخ مفاتيح Supabase

1. روح على **Settings** ← **API**
2. انسخ هذي القيم في ملف `.env.local`:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
   - **anon / public key** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### 4️⃣ تشغيل اللعبة محلياً

```bash
npm run dev
```

افتح [http://localhost:3000](http://localhost:3000) وجرّب اللعبة.

**للتجربة الكاملة:** افتح اللعبة في تبويبين (تبويب عادي وتبويب خاص/incognito) عشان تكون لاعبين مختلفين.

## 🌐 النشر على Vercel

### الطريقة الأسهل (تكامل GitHub):

1. روح على [vercel.com](https://vercel.com) وسجّل دخول بحساب GitHub
2. اضغط **Add New** ← **Project**
3. اختر مستودع `arabic-word-game`
4. **مهم**: قبل تضغط Deploy، اضغط **Environment Variables** وأضف:
   - `NEXT_PUBLIC_SUPABASE_URL` = نفس قيمة Supabase
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = نفس قيمة المفتاح
5. اضغط **Deploy**
6. بعد دقيقتين راح يعطيك رابط زي `https://arabic-word-game.vercel.app`

### تحديث اللعبة بعد التغييرات:

```bash
git add .
git commit -m "تحديث"
git push
```

Vercel يحدّث اللعبة تلقائياً مع كل push 🎉

## 🎯 كيف تشتغل اللعبة

### اللعب الأونلاين:

1. اللاعب الأول يضغط **إنشاء غرفة** → يكتب اسمه ويختار الإعدادات → يجيه كود 4 أرقام
2. يرسل الكود للاعب الثاني (واتساب، رسالة، إلخ)
3. اللاعب الثاني يضغط **دخول بكود** → يدخل الكود → يكتب اسمه
4. تبدأ اللعبة على الجوالين بنفس اللحظة
5. الساعات والكلمات تتزامن تلقائياً

### كلمة غير موجودة في القاموس:

- لما تكتب كلمة مو في القاموس، **الطرف الثاني** هو اللي يقرر يقبلها أو يرفضها من جواله
- الوقت يتوقف لكلا اللاعبين أثناء القرار
- لو قُبلت، تنضاف للقاموس في هذه الجلسة فقط

## ⚙️ التخصيص

### تعديل القاموس:
أضف كلمات في `lib/dictionary.js` ضمن مصفوفة `ARABIC_DICTIONARY`.

### تعديل النهايات الجاهزة:
عدّل مصفوفة `ENDING_PRESETS` في `components/Game.jsx`.

### تعديل سرعات اللعبة:
عدّل مصفوفة `TIME_CONTROLS` في `components/Game.jsx` (غيّر قيمة `minutes`).

## 🔒 ملاحظات الأمان

- اللعبة مفتوحة بدون مصادقة (anonymous play)، أي شخص عنده الكود يقدر يلعب
- الـ `anon key` آمن في الـ frontend (مسموح كشفه)
- الـ Row Level Security مفعّل في الجدول لكن السياسات مفتوحة (مناسبة لاستخدام شخصي/عائلي)
- **لا تستخدم** الـ `service_role key` في الفرونت إند أبداً

### حد الـ Free Tier في Supabase:
- 500 ميجا داتابيس
- 2 جيجا نقل/شهر
- 200 اتصال متزامن (أكثر من كافي للعب الشخصي)

## 🐛 حل المشاكل الشائعة

**اللعبة ما تشتغل لما أفتح الموقع:**
- تأكد إن المتغيرات في `.env.local` صحيحة
- في Vercel، تأكد إنك ضفت Environment Variables في الإعدادات

**الكلمات ما تتزامن بين الجوالين:**
- تأكد إنك شغّلت `supabase-schema.sql` كامل
- بالأخص السطر: `ALTER PUBLICATION supabase_realtime ADD TABLE rooms;`
- جرّب refresh للصفحة

**الكود ينقال "تعذر إنشاء الغرفة":**
- شيك Console (F12) عشان تشوف الخطأ التفصيلي
- غالباً متغيرات البيئة ناقصة أو سكيما مو مشغّلة
